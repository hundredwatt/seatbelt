# Seatbelt Error Analysis Script

This Python script analyzes error output from the seatbelt Go implementation and provides detailed diff for drifted rows.

## Installation

Install the required dependencies:

```bash
pip install -r requirements.txt
```

## Usage

```bash
./analyze_errors.py <config_path> <error_file_path>
```

### Parameters

- `config_path`: Path to the seatbelt configuration YAML file
- `error_file_path`: Path to the JSON error file generated by the seatbelt Go implementation

### Example

```bash
./analyze_errors.py examples/config.yaml tmp/seatbelt-errors-public.data_proof_test-3127687041.json
```

## Features

- **Error Classification**: Automatically classifies errors as:
  - **Source-Only**: Records that exist in source but not in target
  - **Target-Only**: Records that exist in target but not in source  
  - **Drifted**: Records that exist in both but have different values

- **Colorful Output**: Uses colors to highlight different types of discrepancies

- **Detailed Diffs**: For drifted rows, shows field-by-field differences between source and target

- **Database Support**: 
  - PostgreSQL source connections
  - ClickHouse target connections

## Output Format

The script provides a summary table showing error counts and detailed row information with diffs for drifted records, similar to the demo-live validation script.

## Configuration

The script reads the seatbelt configuration file to:
- Parse source (PostgreSQL) and target (ClickHouse) connection strings
- Extract table names and column mappings
- Determine primary key column

## Error File Format

The script expects a JSON file with error records in the format:
```json
[
  {
    "pk": 27,
    "source_signature": 8532021055372091661,
    "target_signature": null
  },
  {
    "pk": 26,
    "source_signature": 8532021055372091661,
    "target_signature": null
  }
]
```
